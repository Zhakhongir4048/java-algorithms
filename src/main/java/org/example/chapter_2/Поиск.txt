Если выбранное значение ключа отсутствует в массиве, приложение проверяет все заполненные ячейки,
а потом сообщает, что значение найти не удалось.

Если массив содержит N элементов, то среднее
количество шагов, необходимых для нахождения элемента, равно N/2. В худшем
случае заданный элемент находится в последней ячейке и для его обнаружения
потребуется N шагов.
Таким образом, время выполнения алгоритма пропорционально количеству
шагов, а поиск в среднем выполняется значительно медленнее (N/2 шагов) вставки
(один шаг).
